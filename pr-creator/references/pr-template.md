# Pull Request Output Template

Use this structure for the generated `PR.md` unless the repository template says otherwise.

## Suggested Title

Write a short title that describes the outcome, not the implementation details.

## Summary

Summarize the change in one to three paragraphs.

## What Changed

List the main themes of the branch, grouped by user-visible or technical impact.

## Testing

Describe any checks, test runs, or manual verification that actually happened.

## Risks and Follow-Ups

List remaining risks, deferred work, and anything that still needs confirmation.

## Template-Aligned Sections

If the repository has a PR template, mirror its headings here and fill them with branch-specific content.

## Checklist

- [ ] The summary matches the branch diff
- [ ] Testing notes are accurate
- [ ] Template-required sections are present
- [ ] No fabricated claims, links, or approvals
